ОТВЕТ ДОЛЖЕН БЫТЬ В ФОРМАТЕ JSON

[Темы для выделения]
1. "Область применения": Краткое описание ГОСТа, где он применяется
2. "Требования к структурным элементам отчета": Словарь, содержащий все структурные элементы отчёта в качестве ключей и список требований к каждому элементу
3. "Правила оформления": В этот пункт должны быть записаны все правила оформления различных инструментов (шрифты, размеры, отступы, цвета, начертания, нумерация и так далее) В данном пункте допускается вложенностью. то есть нужно отдельно рассмотреть оформления для таблиц, рисунков, сносок, формул, ссылок и так далее 

[Рекомендации]
1. Отдельное внимание удели 3 пункту. Этот пункт должен быть развёрнутым,  так как его содержание очень важно.
2. Численные значения такие как размер шрифта, отступы, толщина линий должны быть представлены с использованием единиц измерения и без лишней информации.
3. Ключи должны быть написанны только русскими буквамии, то есть весь отчет должен быть на русском языке (например, "Применение" вместо "application_area").
4. Числовые значения следует хранить в либо в целочисленном, либо в десятичном формате
5. Не повторяй вопрос в ответе.
6. Не изменяй первоначальный текст.
8. Старайся захватить как можно больше численной информации и других кратких характеристик. Они содежрат наиболее важную информацию
9. Следи за правильностью стурктуры JSON. Все скобки и кавычки должны быть закрыты
10. Обращай внивание на условные требования, которые выполняются только в особых случаях

[Структура ответа]
{
    "params": "список всех возможных параметров в виде массива",
    "elements": "список всех возможных элементов ГОСТа. Элементы в данном случае - это мелкие составляющие ГОСТа, а не разделы" (element: краткое наименование, description: описание),
    "gost_params": "связь параметров с конкретными элементами ГОСТа и их значение"
}
